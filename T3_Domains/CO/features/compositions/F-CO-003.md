## SS0. Identity

### 0.1 Purpose
This composition groups all capabilities for maintaining product cost estimates, marking and releasing standard prices, and analyzing cost variances. A cost accountant uses these features to calculate the standard cost of manufactured goods, release prices to materials management, and investigate deviations between planned and actual production costs.

## SS1. Children & Variability Structure

### 1.1 Group Logic
**Group type:** mixed

**Business rationale:** Maintain and Mark/Release are mandatory because a product costing process requires both creating the estimate and activating the standard price. Variance Analysis is optional because it requires production order actuals (from OPS/PPS) — deployments without manufacturing can omit it.

### 1.2 Sub-Groups

| Sub-group | Children | Group type | Rationale |
|---|---|---|---|
| Standard costing | `003-01`, `003-02` | mandatory | Core product costing workflow |
| Variance reporting | `003-03` | optional | Only applicable when production actuals are available |

### 1.3 Rationale for Tree Position
Product Costing consumes cost center overhead rates from F-CO-002 and feeds the profitability composition F-CO-004 with standard COGS values. It also integrates with PPS and OPS for actual cost collection and order settlement.

## SS3. Quality & Testing Guidance
All leaf features MUST be independently testable. Integration tests MUST verify that releasing a cost estimate in F-CO-003-02 publishes a `co.pc.cost-estimate.released` event consumed by co-pa-svc. Variance results MUST reconcile with actual order costs within the same fiscal period.
